# Contributing

Thanks for your interest in `multi-agent-fiction-lab`.

This lab is currently in **v0.1** — actively building. The following kinds of contribution are welcome.

## Open contributions

- **Bug reports** (typos, broken links, factual errors in docs) — open an issue
- **Discussion of methodology** (you tried something different, the lab's claims seem wrong, alternative pattern you've seen) — open an issue or discussion
- **Questions about implementation** — open a discussion
- **Pointing out missing references / related work** — open an issue

## Limited contributions (PRs by invitation)

The lab currently does **not** accept open pull requests for:

- Methodology document edits — substance is maintained by the author during v0.x build
- Case study text — bound to specific source novels under separate IP licensing
- Workflow template SOULs — voice IP belongs to specific authors

If you have a substantial proposal for any of the above, please **open an issue first to discuss**. If we agree it makes sense, we'll invite a PR.

## How to file a good issue

Useful issues have:

- A clear summary in the title
- What you observed (with file path + line if applicable)
- What you expected
- Why the gap matters (especially for methodology disagreements — argument > assertion)

Issues that just say "this is wrong" without an argument will likely be closed asking for context.

## Future opening

When v1.0 lands (target 2026-10), we'll review this policy. The lab is meant to be a methodology share, not a closed system — but during v0.x build we keep the substance tightly maintained.

## Code of conduct

Standard expectations: respectful discussion, no harassment, signal over noise. We reserve the right to moderate discussions that don't meet these standards.
